## 一、Mod 加载系统架构

### 1.1 加载流程

游戏启动时 `GameManager` 触发加载：

```
GameManager 初始化
  │
  ├─→ ModManager.LoadMods()
  │     ├─ 扫描 Mods/ 文件夹
  │     ├─ 解析每个子目录的 ModInfo.xml
  │     ├─ TFP_Harmony 强制排到最前
  │     ├─ 加载 DLL 程序集
  │     └─ InitModCode() → 反射查找 IModApi 实现
  │
  ├─→ ModManager.LoadPatchStuff()
  │     ├─ LoadUiAtlases() → UI 图集
  │     └─ LoadLocalizations() → 本地化文本
  │
  ├─→ WorldStaticData.loadStaticData()
  │     └─ XmlPatcher.LoadAndPatchConfig() → 普通 XML 补丁
  │
  └─→ XUi.SetXmlsForUi()
        └─ PatchAndLoadXuiXml() → XUi XML 补丁
```

### 1.2 Mod 扫描路径

游戏从两个位置扫描 Mods 文件夹：

| 路径 | 说明 |
|------|------|
| `游戏根目录/Mods/` | 主要路径（推荐） |
| `用户数据目录/Mods/` | 备用路径 |

文件夹按名称字母排序加载。名为 `TFP_Harmony` 的 Mod 会被强制移到加载列表最前面。

### 1.3 Mod 加载状态

| 状态 | 含义 |
|------|------|
| `Success` | 加载成功 |
| `NotAntiCheatCompatible` | 不兼容反作弊 |
| `SkippedDueToAntiCheat` | 因反作弊跳过 |
| `DuplicateModName` | 名称重复 |
| `Failed` | 通用失败 |

## 二、ModInfo.xml 规范

### 2.1 文件格式（V2，唯一支持格式）

```xml
<?xml version="1.0" encoding="UTF-8"?>
<xml>
    <Name value="MyMod" />                    <!-- 必填，仅 [0-9a-zA-Z_-] -->
    <DisplayName value="我的Mod" />            <!-- 必填，非空 -->
    <Version value="1.0.0" />                  <!-- 可选，合法 Version -->
    <Description value="Mod描述" />            <!-- 可选 -->
    <Author value="作者名" />                  <!-- 可选 -->
    <Website value="https://example.com" />    <!-- 可选 -->
    <SkipWithAntiCheat value="true" />         <!-- 可选，布尔值 -->
</xml>
```

### 2.2 注意事项

- **V1 格式已废弃**：根元素为 `<ModInfo>` 的旧格式会被直接拒绝
- **Name 必须唯一**：重名 Mod 会被忽略
- **Name 正则校验**：`^[0-9a-zA-Z_\-]+$`
- 文件必须命名为 `ModInfo.xml`，放在 Mod 根目录

### 2.3 纯 XML Mod 目录结构

```
Mods/
└── MyMod/
    ├── ModInfo.xml           ← 必须有
    └── Config/               ← XML 配置补丁目录
        ├── buffs.xml         ← 修改 Buff 定义
        ├── items.xml         ← 修改物品定义
        ├── blocks.xml        ← 修改方块定义
        ├── recipes.xml       ← 修改配方
        ├── progression.xml   ← 修改技能树
        ├── entityclasses.xml ← 修改实体属性
        ├── quests.xml        ← 修改任务定义
        ├── XUi/              ← XUi 界面补丁（游戏内UI）
        │   ├── windows.xml
        │   ├── controls.xml
        │   ├── styles.xml
        │   └── xui.xml
        ├── XUi_Menu/         ← XUi 界面补丁（主菜单UI）
        │   ├── windows.xml
        │   ├── controls.xml
        │   ├── styles.xml
        │   └── xui.xml
        └── Localization.txt  ← 本地化文本
```

## 三、XML 配置补丁系统

### 3.1 补丁文件路径匹配

游戏通过 `XmlPatcher.LoadAndPatchConfig` 加载配置时，会检查每个 Mod 的 `Config/` 目录下是否有同名文件：

| 配置名 | 游戏原始路径 | Mod 补丁路径 |
|--------|-------------|-------------|
| `buffs.xml` | `Data/Config/buffs.xml` | `Mods/MyMod/Config/buffs.xml` |
| `items.xml` | `Data/Config/items.xml` | `Mods/MyMod/Config/items.xml` |
| `XUi/windows.xml` | `Data/Config/XUi/windows.xml` | `Mods/MyMod/Config/XUi/windows.xml` |
| `XUi/controls.xml` | `Data/Config/XUi/controls.xml` | `Mods/MyMod/Config/XUi/controls.xml` |
| `XUi_Menu/windows.xml` | `Data/Config/XUi_Menu/windows.xml` | `Mods/MyMod/Config/XUi_Menu/windows.xml` |

**关键**：路径必须完全匹配，包括子目录。XUi 文件必须放在 `Config/XUi/` 或 `Config/XUi_Menu/` 下。

### 3.2 补丁文件格式

补丁文件根元素必须是 `<configs>`：

```xml
<configs>
    <append xpath="/buffs">
        <!-- 追加内容 -->
    </append>

    <set xpath="/buffs/buff[@name='god']/effect_group/passive_effect[@name='CarryCapacity']/@value">
        180
    </set>

    <remove xpath="/buffs/buff[@name='someBuff']"/>
</configs>
```

### 3.3 补丁操作速查

| 操作 | 说明 | 示例 |
|------|------|------|
| `append` | 在匹配节点内追加子元素 | `<append xpath="/buffs">...</append>` |
| `prepend` | 在匹配节点内前置子元素 | `<prepend xpath="/buffs">...</prepend>` |
| `insertAfter` | 在匹配节点后插入 | `<insertAfter xpath="/buffs/buff[1]">...</insertAfter>` |
| `insertBefore` | 在匹配节点前插入 | `<insertBefore xpath="/buffs/buff[1]">...</insertBefore>` |
| `set` | 替换匹配元素的内容或属性 | `<set xpath=".../@attr">新值</set>` |
| `remove` | 删除匹配的元素 | `<remove xpath="..."/>` |
| `setattribute` | 设置/覆盖属性值 | `<setattribute xpath="..." name="attr">值</setattribute>` |
| `removeattribute` | 删除属性 | `<removeattribute xpath="..." name="attr"/>` |

### 3.4 XPath 常用语法

```xml
<!-- 按属性匹配 -->
/buffs/buff[@name='god']

<!-- 匹配属性值并修改 -->
<set xpath="/buffs/buff[@name='god']/effect_group/passive_effect[@name='CarryCapacity']/@value">180</set>

<!-- 匹配子元素 -->
/windows/window[@name='windowBackpack']/panel[@name='content']/grid[@name='inventory']

<!-- 使用逻辑运算 -->
/buffs/buff[@name='buffA' or @name='buffB']
```

## 四、XUi 界面 Mod 开发

### 4.1 XUi 加载流程

```
XUi.SetXmlsForUi()
  │
  ├─ 主 UI (isPrimaryUI=true):
  │   ├─ XUi_Common/styles.xml
  │   ├─ XUi_Common/controls.xml
  │   ├─ XUi_Menu/styles.xml
  │   ├─ XUi_Menu/controls.xml
  │   ├─ XUi_Menu/windows.xml     ← 主菜单窗口
  │   └─ XUi_Menu/xui.xml
  │
  └─ 非 主 UI:
      ├─ XUi/styles.xml
      ├─ XUi/controls.xml
      ├─ XUi/windows.xml           ← 游戏内窗口（背包在这里！）
      └─ XUi/xui.xml
```

**注意**：背包窗口 `windowBackpack` 定义在 `XUi/windows.xml` 中，补丁必须放在 `Config/XUi/windows.xml`。

### 4.2 XUi 控件层级

```
XUiController
  ├── XUiC_BackpackWindow     ← 背包窗口控制器
  ├── XUiC_ItemStackGrid      ← 物品格子网格基类
  │   ├── XUiC_Backpack       ← 背包格子网格
  │   └── XUiC_VariableHeightGrid
  └── XUiC_ItemStack          ← 单个格子控制器
```

### 4.3 窗口定义结构

```xml
<window name="windowBackpack" width="606" height="349" panel="Center"
        controller="BackpackWindow" cursor_area="true">

    <!-- 头部面板 -->
    <panel name="header" height="43" depth="1" backgroundspritename="ui_game_panel_header">
        <sprite name="windowIcon" .../>
        <label name="windowName" .../>
        <!-- 操作按钮 -->
    </panel>

    <!-- 内容面板 -->
    <panel name="content" width="100%" height="303" pos="0,-46" disableautobackground="true">
        <sprite name="backgroundMain" .../>  <!-- 背景 -->
        <grid name="inventory" rows="5" cols="9" cell_width="67" cell_height="67"
              controller="Backpack" repeat_content="true">
            <backpack_item_stack name="0"/>  <!-- 格子模板 -->
        </grid>
    </panel>
</window>
```

### 4.4 Grid 网格属性

| 属性 | 说明 | 默认值 |
|------|------|--------|
| `rows` | 行数 | - |
| `cols` | 列数 | - |
| `cell_width` | 格子宽度(px) | - |
| `cell_height` | 格子高度(px) | - |
| `repeat_content` | 是否重复生成格子 | false |
| `arrangement` | 排列方式 | horizontal |
| `controller` | 控制器类名 | - |

### 4.5 常用 XUi 元素

| 元素 | 说明 | 关键属性 |
|------|------|---------|
| `<sprite>` | 图片/色块 | `sprite`, `color`, `width`, `height`, `type="sliced"` |
| `<filledsprite>` | 填充进度条 | `fill="{变量}"`, `color` |
| `<label>` | 文本标签 | `text`, `font_size`, `color`, `effect="outline"` |
| `<button>` | 按钮 | `sprite`, `style`, `sound` |
| `<rect>` | 矩形容器 | `pos`, `width`, `height`, `visible` |
| `<panel>` | 面板 | `height`, `depth`, `backgroundspritename` |
| `<grid>` | 网格 | `rows`, `cols`, `cell_width`, `cell_height` |
| `<texture>` | 纹理 | `material`, `size` |

### 4.6 颜色格式

XUi 使用 `R,G,B,A` 格式，每个分量 0-255：

```xml
color="0,220,255,255"     <!-- 青色，不透明 -->
color="8,18,38,220"       <!-- 深蓝，半透明 -->
color="255,0,0,255"       <!-- 红色，不透明 -->
```

### 4.7 数据绑定

```xml
<!-- 绑定变量 -->
text="{itemcount}"                    <!-- 物品数量 -->
text="{usedslots}/180"                <!-- 已用/总槽位 -->
fill="{encumbrance}"                  <!-- 负重比例 -->
visible="{isfavorite}"                <!-- 是否收藏 -->
color="{durabilitycolor}"             <!-- 耐久颜色 -->

<!-- 条件表达式 -->
visible="{# ishovered and islocked}"  <!-- C# 表达式 -->
color="{# userlockedslot ? '0,220,255' : '255,255,255' }"  <!-- 三元运算 -->
```

## 五、实战案例：180格科幻风大背包 Mod

### 5.1 设计规格

| 项目 | 原版 | Mod |
|------|------|-----|
| 格子数 | 45 (9x5) | 180 (20x9) |
| 格子尺寸 | 67x67 | 45x45 |
| 窗口宽 | 606 | 920 |
| 窗口高 | 349 | 470 |
| 负重 | 有惩罚 | 无惩罚 |
| 主题 | 默认 | 科幻青蓝 |

### 5.2 文件结构

```
Mods/BigBackpack180/
├── ModInfo.xml
└── Config/
    ├── buffs.xml              ← 数据层
    └── XUi/
        ├── windows.xml        ← 窗口布局
        └── controls.xml       ← 格子模板
```

### 5.3 buffs.xml — 数据层

通过被动效果修改背包大小和负重：

```xml
<configs>
    <!-- 添加大背包 buff -->
    <append xpath="/buffs">
        <buff name="buffBigBackpack180" hidden="true" remove_on_death="false">
            <stack_type value="ignore"/>
            <update_rate value="5"/>
            <effect_group>
                <passive_effect name="BagSize" operation="base_set" value="180"/>
                <passive_effect name="CarryCapacity" operation="base_set" value="180"/>
            </effect_group>
        </buff>
    </append>

    <!-- 通过常驻状态检查 buff 自动触发 -->
    <append xpath="/buffs/buff[@name='buffStatusCheck01']/effect_group">
        <triggered_effect trigger="onSelfBuffUpdate" action="AddBuff" buff="buffBigBackpack180">
            <requirement name="!HasBuff" buff="buffBigBackpack180"/>
        </triggered_effect>
    </append>
</configs>
```

**关键点**：
- `BagSize`：背包数组总大小（UI 显示的格子数）
- `CarryCapacity`：可解锁的格子数（超出部分显示锁定）
- 两者都设为 180 = 全部解锁无负重
- 通过 `buffStatusCheck01`（玩家常驻 buff）自动添加，确保始终生效

### 5.4 windows.xml — 窗口布局

替换整个背包窗口，添加科幻风 UI：

```xml
<configs>
    <remove xpath="/windows/window[@name='windowBackpack']"/>

    <append xpath="/windows">
        <window name="windowBackpack" width="920" height="470"
                panel="Center" controller="BackpackWindow" cursor_area="true">

            <!-- 头部：深蓝底 + 青色标题 + 容量显示 -->
            <panel name="header" height="43" depth="1" backgroundspritename="ui_game_panel_header">
                <sprite depth="0" pos="0,0" width="920" height="43" color="8,18,38,220" type="sliced"/>
                <sprite depth="1" pos="0,-1" width="920" height="2" color="0,220,255,200" type="sliced"/>
                <!-- ... 标题、容量条、按钮 ... -->
            </panel>

            <!-- 内容区：20列x9行网格 -->
            <panel name="content" width="100%" height="420" pos="0,-46" disableautobackground="true">
                <sprite depth="0" pos="0,0" width="920" height="420" color="5,12,28,200" type="sliced"/>
                <sprite depth="8" name="backgroundMain" ... color="0,160,200,120" type="sliced" fillcenter="false"/>
                <grid depth="10" name="inventory" rows="9" cols="20"
                      pos="8,-8" cell_width="45" cell_height="45"
                      controller="Backpack" repeat_content="true">
                    <backpack_item_stack name="0"/>
                </grid>
            </panel>
        </window>
    </append>
</configs>
```

### 5.5 controls.xml — 格子模板

替换 `backpack_item_stack` 模板，适配 45x45 小格子：

```xml
<configs>
    <remove xpath="/controls/backpack_item_stack"/>

    <append xpath="/controls">
        <backpack_item_stack controller="ItemStack" ...>
            <rect ...>
                <!-- 青色发光边框 -->
                <sprite name="backgroundMain" color="0,140,180,100" width="43" height="43"/>
                <!-- 深蓝底色 -->
                <sprite name="background" color="8,20,45,220" width="39" height="39"/>
                <!-- 物品图标、耐久条、数量标签等，全部按比例缩小 -->
                ...
            </rect>
        </backpack_item_stack>
    </append>
</configs>
```

### 5.6 踩坑记录

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| windows.xml 补丁不生效 | 放在 `Config/windows.xml` | 必须放在 `Config/XUi/windows.xml` |
| XUi_Menu 补丁报错 | 背包窗口不在主菜单 UI 中 | 背包窗口在 `XUi/windows.xml`，不是 `XUi_Menu/windows.xml` |
| buffs.xml xpath 找不到 | `buffStatusUpdate` 不存在 | 使用 `buffStatusCheck01` |
| 格子显示不全 | 数据层和 UI 层不匹配 | BagSize/CarryCapacity 必须与 grid rows*cols 一致 |

## 六、调试与排错

### 6.1 查看游戏日志

日志路径：`%USERPROFILE%\AppData\LocalLow\The Fun Pimps\7 Days To Die\Player.log`

```powershell
# 查看 Mod 加载情况
Select-String -Path $env:USERPROFILE\AppData\LocalLow\"The Fun Pimps"\"7 Days To Die"\Player.log -Pattern "MODS"

# 查看 XML 补丁错误
Select-String -Path $env:USERPROFILE\AppData\LocalLow\"The Fun Pimps"\"7 Days To Die"\Player.log -Pattern "did not apply"

# 查看特定 Mod 的所有日志
Select-String -Path $env:USERPROFILE\AppData\LocalLow\"The Fun Pimps"\"7 Days To Die"\Player.log -Pattern "BigBackpack180"
```

### 6.2 常见错误

| 日志关键词 | 原因 | 解决 |
|-----------|------|------|
| `did not apply` | XPath 没匹配到节点 | 检查 XPath 是否正确，检查文件路径 |
| `does not contain a ModInfo.xml` | 缺少 ModInfo.xml | 创建 V2 格式的 ModInfo.xml |
| `legacy format, V2 required` | 使用了 V1 格式 | 改为 V2 格式（根元素 `<xml>`） |
| `DuplicateModName` | Mod 名称重复 | 修改 Name 值 |
| `AntiCheat needs to be disabled` | 反作弊不兼容 | 设置 SkipWithAntiCheat="true" |

### 6.3 XML 补丁调试技巧

1. **先确认原始 XML 结构**：查看 `Data/Config/` 下的原始文件
2. **确认补丁路径**：XUi 文件必须放在正确的子目录
3. **逐步测试**：先测试简单的 `set` 操作，再测试复杂的 `remove` + `append`
4. **查看日志**：`did not apply` 会告诉你具体哪行 XPath 失败

## 附录B：XML 补丁操作速查

```xml
<configs>
    <!-- 1. 追加子元素 -->
    <append xpath="/buffs">
        <buff name="myBuff">...</buff>
    </append>

    <!-- 2. 修改属性值 -->
    <set xpath="/buffs/buff[@name='god']/effect_group/passive_effect[@name='CarryCapacity']/@value">180</set>

    <!-- 3. 删除元素 -->
    <remove xpath="/buffs/buff[@name='unwantedBuff']"/>

    <!-- 4. 替换整个元素内容 -->
    <set xpath="/windows/window[@name='windowBackpack']">
        <window name="windowBackpack" ...>...</window>
    </set>

    <!-- 5. 设置属性 -->
    <setattribute xpath="/windows/window[@name='windowBackpack']" name="height">470</setattribute>

    <!-- 6. 条件插入 -->
    <insertAfter xpath="/buffs/buff[last()]">
        <buff name="newBuff">...</buff>
    </insertAfter>
</configs>
```
