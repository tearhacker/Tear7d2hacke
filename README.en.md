## 📖 Overview

**TearHeart Trainer** is a feature-rich, professionally engineered modification tool for *7 Days to Die*, built on **Unity IMGUI** with a fully real-time rendered interface. The tool is structured around three core tabs — **Visuals (ESP)**, **Aimbot**, and **Misc (Memory Cheats)** — providing a complete suite of visual enhancements, targeting assistance, and gameplay modifications.

> 🌐 **Official Distribution Channel**: All authentic, up-to-date builds are released exclusively through the official portal: **[teargamestorem.top](http://teargamestorem.top/)**

## ✨ Core Features

### 🎯 1. Visuals Tab (ESP / Wallhack System)

A complete entity highlighting system rendered through Unity IMGUI overlays.

**Master Controls**
- **One-Click Enable** — instantly activates all ESP categories, snaplines, names, distances, healthbars, skeleton ESP and 3D box mode.
- **Global Outline Modes** — `Off` / `Solid Color` / `Material Outline`.
- **Global Box Type** — `None` / `2D Screen Box` / `3D World Box`.

**Entity ESP Table** (Players · Enemies · NPCs · Animals)

| Column | Description |
|--------|-------------|
| Master Toggle | Enable ESP for the category |
| Snapline | Tracer line from screen to entity |
| Name | Render entity name |
| Distance | Render distance to player |
| Healthbar | Render entity health |

- **Skeleton ESP** — Dedicated bone-overlay rendering for enemies.
- **Alternating row shading** with dimmed rows when disabled, for clear readability.

**Item / Container ESP** (6 categories: Loot · Crates · Vehicles · Loot Bags · Storage · Ore)
- Single-toggle activation that auto-enables name + distance display.
- Toggling off preserves user-defined name/distance state.

**Player HUD Card**
- Master toggle disables the entire HUD block.
- 4×3 information grid:
  - **Row 1**: Level · XP Progress · Skill Points
  - **Row 2**: Core Temperature · Game Stage · World Coordinates
  - **Row 3**: Blood Moon Countdown · Kill Stats · Death Count
  - **Row 4**: Current Survival Time

### 🎯 2. Aimbot Tab (Targeting Assistance)

Two independent aim systems with isolated configuration profiles.

#### 🔮 Mouse Aimbot (Visible Cursor-Driven)

| Group | Options |
|-------|---------|
| **Trigger** | Always-On toggle · Custom hotkey binding (live key listener) |
| **Targets** | Enemies · Other Players |
| **Bone** | Head / Neck / Chest / Pelvis (segmented selector) |
| **Behavior** | Target Lock · Visible-Only · Max Distance slider |
| **FOV** | FOV Limit toggle · FOV Size · Visualize FOV Circle |
| **Smoothing** | Smoothing factor — anti-snap detection mitigation |

#### 🌑 Silent Aimbot (Memory-Based Bullet Redirection)

> *Internally referred to as "Bullet Tracking" — operates by modifying in-memory ballistic targeting coordinates rather than moving the cursor.*

| Group | Options |
|-------|---------|
| **Targets** | Enemies · Other Players |
| **Bone** | Head / Neck / Chest / Pelvis |
| **Behavior** | Target Lock · Visible-Only · Max Tracking Distance |

Each card uses dynamic height calculation that expands/collapses with toggle states.
